# 洞察配置优化说明

> 🎯 让洞察数据完全对应图表数据进行分析

**优化日期**: 2026-04-08  
**版本**: 3.1.0-dynamic

---

## 📋 优化前的问题

### ❌ 问题 1：SKILL.md 写死固定统计表

**之前**：
```markdown
## 📊 已知数据结构

你的输入包含以下统计表：

| Sheet 名称 | 内容说明 |
|-----------|---------|
| 核心 KPI | 整体经营指标 |
| 销售员业绩 | 9 名销售员业绩 |
| 产品占比 | 6 类产品销售 |
| ... (固定 10 个表) |
```

**问题**：新增图表后，AI 不知道新的统计表，无法分析。

---

### ❌ 问题 2：洞察与图表不匹配

**之前**：
- 固定输出 10 条洞察
- 对应固定的 10 个图表
- 新增图表后，洞察数量不匹配

---

### ❌ 问题 3：手动更新 SKILL.md

**之前**：
- 每次新增图表都要手动修改 SKILL.md
- 容易遗漏或出错
- 维护成本高

### ✅ 方案 2：自动检测图表配置

**修改文件**: `scripts/ai/insight_generator.py`

**新增功能**:
```python
def _load_charts_config(self) -> Optional[Dict]:
    """动态加载图表配置"""
    placeholders_path = os.path.join(self.base_dir, 'templates', 'placeholders.json')
    if os.path.exists(placeholders_path):
        with open(placeholders_path, 'r', encoding='utf-8') as f:
            config = json.load(f)
            return config.get('placeholders', {}).get('charts', {})
    return None
```

**效果**:
- AI 知道实际的图表数量和配置
- 洞察数量 = 图表数量
- 每条洞察对应一个图表

## 🚀 使用示例

### 新增"客户城市分析"图表

**步骤 1**: 在 `stats_rules.json` 中添加统计规则
```json
"客户城市分析": {
  "description": "客户属性与城市交叉分析",
  "type": "ranking",
  "enabled": true,
  "group_by": ["客户属性", "城市"],
  "metrics": [
    {"field": "销售额", "agg": "sum", "alias": "总销售额"}
  ]
}
```

**步骤 2**: 在 `placeholders.json` 中添加图表配置
```json
"CHART:customer_city": {
  "data_source": "客户城市分析",
  "chart_type": "bar_horizontal",
  "x_field": "总销售额",
  "y_field": "城市",
  "title": "客户属性与城市销售分析"
}
```

**步骤 3**: 运行 `Run.bat`
```
自动检测图表配置变化
→ 重新生成 SKILL.md
→ 生成 7 条洞察（原 6 条 + 新增 1 条）
→ 填充 PPT
```

**结果**:
- ✅ SKILL.md 包含"客户城市分析"统计表
- ✅ AI 生成对应的洞察
- ✅ PPT 显示新图表和洞察

with open(placeholders_path, 'rb') as f:

### 3. 图表 - 洞察映射

```python
def _build_user_prompt(self, data_context: str) -> str:
    """构建用户 Prompt"""
    charts_config = self._load_charts_config()
    
    chart_info = "\n\n【图表配置】\n"
    for i, (chart_key, chart_cfg) in enumerate(charts_config.items(), 1):
        chart_info += f"{i}. {chart_key}: {chart_cfg.get('title', '')} (数据源：{chart_cfg.get('data_source', '')})\n"
    
    return f"""请根据以下销售数据，生成符合上述规范的商业洞察。

数据摘要:
{data_context}
{chart_info}

记住：
1. JSON 数组中每条只包含纯洞察内容，不要任何前缀
2. 洞察数量 = 图表数量（{len(charts_config)}个）
3. 每条洞察对应一个图表，按顺序分析"""
```

**优点**: AI 清楚知道要生成多少条洞察

## 🎯 最佳实践
