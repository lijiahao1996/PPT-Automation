## 1. 系统概述

本系统通过配置文件实现 PPT 报告的自动生成，无需修改 Python 代码即可新增图表和统计维度。

### 1.1 核心优势

- **配置化**：所有统计规则和图表配置都在 JSON 文件中
- **零硬编码**：业务数据完全从配置读取
- **易扩展**：新增图表只需 5-10 分钟
- **灵活性强**：统计配置和展示配置分离

### 1.2 文件结构

```
templates/
├── stats_rules.json           ← 统计规则配置
├── placeholders.json          ← PPT 图表配置
├── 配置指南.docx              ← 本文档
├── PPT 新增图表指南.md        ← PPT 配置说明
└── 销售分析报告_标准模板.pptx ← PPT 模板
```

## 2. 配置文件说明

系统使用两个核心配置文件：

| 配置文件 | 用途 | 修改频率 |
|---------|------|---------|
| stats_rules.json | 定义统计规则，生成 Excel 数据 | 新增统计维度时 |
| placeholders.json | 定义图表类型和 PPT 位置 | 新增图表时 |

**配置流程**:
1. 在 stats_rules.json 中配置统计规则
2. 运行生成 Excel 数据
3. 在 placeholders.json 中配置图表
4. 在 PPT 模板中添加占位符
5. 运行生成 PPT

## 3. stats_rules.json 配置详解

此文件定义统计规则，决定生成哪些 Excel Sheet。

### 3.1 文件结构

```json
{
  "version": "1.0",
  "stats_sheets": {
    "核心 KPI": { ... },
    "销售员业绩": { ... }
  },
  "global_settings": { ... }
}
```

### 3.2 统计类型

| 类型 | type 值 | 用途 |
|------|--------|------|
| 核心 KPI | kpi | 汇总指标 |
| 排名统计 | ranking | 销售员/城市排名 |
| 占比分析 | composition | 产品占比 |
| 对比分析 | comparison | 新老客对比 |
| 趋势分析 | trend | 月度趋势 |
| 分布分析 | distribution | 星期分布 |
| 矩阵分析 | matrix | 销售员 - 产品 |
| 异常检测 | outlier | 异常订单 |

### 3.3 配置示例

```json
"销售员业绩": {
  "description": "销售员业绩排名",
  "type": "ranking",
  "enabled": true,
  "group_by": ["销售员"],
  "metrics": [
    {"field": "销售额", "agg": "sum", "alias": "总销售额"}
  ],
  "sort_by": "总销售额",
  "sort_asc": false
}
```

### 3.4 参数说明

| 参数 | 说明 | 是否必需 |
|------|------|---------|
| type | 统计类型 | 必需 |
| enabled | 是否启用（true/false） | 必需 |
| group_by | 分组字段数组 | 视类型而定 |
| metrics | 统计指标数组 | 必需 |
| metrics[].field | 原始字段名 | 必需 |
| metrics[].agg | 聚合函数 | 必需 |
| metrics[].alias | 结果列名 | 必需 |
| sort_by | 排序字段 | 可选 |

### 3.5 聚合函数

| 函数 | agg 值 | 说明 |
|------|-------|------|
| 求和 | sum | 汇总 |
| 计数 | count | 统计数量 |
| 平均 | mean | 计算平均值 |
| 最大 | max | 最大值 |
| 最小 | min | 最小值 |
| 中位数 | median | 中位数 |

## 4. placeholders.json 配置详解

此文件定义图表配置，决定 PPT 中显示哪些图表。

### 4.1 文件结构

```json
{
  "version": "3.0",
  "placeholders": {
    "charts": {
      "CHART:sales_by_person": { ... }
    },
    "insights": { ... },
    "text": { ... }
  }
}
```

### 4.2 图表类型

| 图表 | chart_type 值 | 需要字段 |
|------|--------------|---------|
| 横向条形图 | bar_horizontal | x_field, y_field |
| 纵向柱状图 | bar_vertical | x_field, y_field |
| 环形饼图 | pie | category_field, value_field |
| 多列柱状图 | column_clustered | category_field, series |
| 折线图 | line | x_field, y_field |
| 热力图 | heatmap | index_field, columns |

### 4.3 配置示例

```json
"CHART:customer_city": {
  "description": "客户 - 城市交叉分析",
  "data_source": "客户 - 城市分析",
  "chart_type": "bar_horizontal",
  "x_field": "总销售额",
  "y_field": "城市",
  "title": "客户属性与城市销售分析"
}
```

### 4.4 参数说明

| 参数 | 说明 | 是否必需 |
|------|------|---------|
| data_source | 数据源（Excel Sheet 名） | 必需 |
| chart_type | 图表类型 | 必需 |
| title | 图表标题 | 必需 |
| x_field | X 轴字段 | 视类型而定 |
| y_field | Y 轴字段 | 视类型而定 |
| category_field | 分类字段 | 视类型而定 |
| series | 数据系列数组 | 视类型而定 |
| description | 说明文字 | 可选 |
| slide_index | 页码（仅文档） | 可选 |

**注意**: slide_index 参数仅用于文档说明，不影响实际图表位置。图表位置由 PPT 模板中的占位符决定。

## 5. PPT 模板配置说明

PPT 模板文件：`销售分析报告_标准模板.pptx`

### 5.1 占位符类型

| 类型 | 格式 | 用途 |
|------|------|------|
| 图表占位符 | [CHART:xxx] | 自动插入图表 |
| 文本占位符 | [TEXT:xxx] | 替换标题等文本 |
| 洞察占位符 | {{INSIGHT:xxx}} | AI 生成的洞察 |
| 表格占位符 | [TABLE:xxx] | 动态表格 |

### 5.2 添加占位符方法

**方法 1: 文本框（推荐）**
1. 插入 → 文本框
2. 输入：[CHART:customer_city]
3. 调整大小和位置

**方法 2: 形状名称**
1. 插入 → 形状
2. 右键 → 大小和位置
3. 名称改为：CHART:customer_city

### 5.3 注意事项

- **占位符名称**必须与 placeholders.json 中的配置一致
- **复制幻灯片**会导致形状内部名称不变，建议重新创建
- **文本框占位符**比形状名称更可靠

## 6. 新增图表完整流程

### 6.1 前提条件

确保原始数据（帆软销售明细.xlsx）中包含新图表所需的字段。

### 6.2 步骤 1: 配置统计规则

**文件**: `templates/stats_rules.json`

1. 打开 stats_rules.json
2. 在 stats_sheets 中添加新配置
3. 保存文件

**示例**:
```json
"客户 - 城市分析": {
  "type": "ranking",
  "enabled": true,
  "group_by": ["客户属性", "城市"],
  "metrics": [
    {"field": "销售额", "agg": "sum", "alias": "总销售额"}
  ]
}
```

### 6.3 步骤 2: 生成统计数据

**运行**: `Run.bat`

**检查**: output/销售统计汇总.xlsx 中是否生成了新的 Sheet

### 6.4 步骤 3: 配置图表

**文件**: `templates/placeholders.json`

1. 打开 placeholders.json
2. 在 charts 中添加新配置
3. 保存文件

**示例**:
```json
"CHART:customer_city": {
  "data_source": "客户 - 城市分析",
  "chart_type": "bar_horizontal",
  "x_field": "总销售额",
  "y_field": "城市",
  "title": "客户属性与城市销售分析"
}
```

### 6.5 步骤 4: 添加 PPT 占位符

**文件**: `templates/销售分析报告_标准模板.pptx`

1. 打开 PPT 模板
2. 找到要放置图表的页面
3. 插入文本框
4. 输入：[CHART:customer_city]
5. 保存 PPT

### 6.6 步骤 5: 测试验证

**运行**: `Run.bat`

**检查**: 生成的 PPT 中是否显示新图表

### 6.7 时间估算

| 步骤 | 时间 |
|------|------|
| 配置统计规则 | 2 分钟 |
| 生成统计数据 | 1 分钟 |
| 配置图表 | 2 分钟 |
| 添加 PPT 占位符 | 2 分钟 |
| 测试验证 | 3 分钟 |
| **总计** | **10 分钟** |

## 7. 常见问题解答

### Q1: 新增图表后不显示？

**A**: 检查以下几点：
1. stats_rules.json 中 enabled 是否为 true
2. placeholders.json 中 data_source 是否与 Excel Sheet 名一致
3. PPT 模板中是否有对应的占位符
4. 占位符格式是否正确（[CHART:xxx]）

### Q2: 图表显示错误的数据？

**A**: 检查字段名是否与 Excel 列名完全一致，包括空格和大小写。

### Q3: 如何禁用某个统计？

**A**: 在 stats_rules.json 中设置 "enabled": false，不要删除配置。

### Q4: 可以删除某个图表吗？

**A**: 可以，在 placeholders.json 中删除对应配置，或在 PPT 模板中删除占位符。

### Q5: slide_index 不生效？

**A**: slide_index 仅用于文档说明，不影响实际位置。图表位置由 PPT 模板中的占位符决定。

### Q6: 运行时提示文件被占用？

**A**: 关闭所有打开的 Excel 和 PPT 文件后重新运行。

### Q7: 如何检查 PPT 占位符？

**A**: 运行脚本：`python scripts\check_ppt_placeholders.py`
